路由、SDN、P4 与 ns-3 参考资料Routing, SDN, P4, and ns-3 references

01 · 路由协议与多路径01 · Routing protocols & multipath

OSPF、BGP 与 ECMPOSPF, BGP, and ECMP

  • RFC 2328 — OSPF Version 2：链路状态数据库、最短路径计算、邻接关系和 Flooding 行为，是 OSPF 基线的重要依据。the link-state database, shortest-path computation, adjacencies, and flooding — the basis for the OSPF baseline.
  • RFC 5340 — OSPF for IPv6：补充 IPv6 环境下的 OSPFv3 机制，帮助区分协议版本差异。OSPFv3 for IPv6, useful for separating protocol-version differences.
  • RFC 4271 — Border Gateway Protocol 4：BGP 路由传播和决策框架。CaelumRoute 第一阶段不尝试替代跨自治系统 BGP。BGP route propagation and decisions. The first stage does not attempt to replace inter-AS BGP.
  • RFC 2992 — Equal-Cost Multi-Path：ECMP 哈希与路径分配分析，作为多路径基线背景。ECMP hashing and path allocation, background for the multipath baseline.

02 · 软件定义网络与可编程数据平面02 · SDN & programmable data planes

OpenFlow、P4 与控制器接口OpenFlow, P4, controller interfaces

  • Open Networking Foundation：SDN 与 OpenFlow 生态的官方组织入口。official entry point for the SDN and OpenFlow ecosystem.
  • P4 Language Consortium：P4 语言、架构和相关规范的官方入口；当前没有硬件实现承诺。official home of the P4 language and specifications; no hardware-implementation promise yet.
  • P4 Language Specification：P4 语言规范仓库，用于核对语义和版本信息。the P4 spec repository, for checking semantics and versions.

范围说明：Scope note: OpenFlow 和 P4 目前是候选执行环境。OpenFlow and P4 are candidate execution environments.

04 · 网络测量与时间同步04 · Measurement & timing

遥测数据的解释边界The limits of telemetry data

链路时延、丢包率和队列状态并不是天然准确的“真实值”。采样周期、探测流量、设备计数器刷新周期、时钟同步和路径变化都会影响观测。项目在引用遥测资料时，将优先说明：Link delay, loss, and queue state are not inherently exact "ground truth." Sampling period, probe traffic, counter refresh, clock sync, and path changes all affect observations. When citing telemetry, the project states:

  • 指标是主动测量、被动测量还是仿真直接读取；whether the metric is active, passive, or read directly from simulation;
  • 测量点位于发送端、接收端、中间设备还是控制器；whether the vantage point is sender, receiver, mid-path, or controller;
  • 数据时间戳和有效期如何定义；how timestamps and validity windows are defined;
  • 缺失、重复、延迟到达和异常数据如何处理；how missing, duplicate, late, or anomalous data is handled;
  • 测量本身产生的网络与计算开销。the network and compute overhead of measurement itself.
