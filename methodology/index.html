研究方法Research Methodology

CATS 拓扑采样与自适应路由评估方法CATS topology sampling & adaptive-routing evaluation

本页将 CaelumRoute 拆分为可验证的问题、变量、流程和评价标准，作为实验实现与结果发布的依据。This page decomposes CaelumRoute into testable questions, variables, procedures, and criteria — the basis for the implementation and the eventual results release.

01 · 方法目标01 · Goal

把“更快路由”转化为可检验问题Turning "faster routing" into testable questions

研究不会只比较平均端到端时延，而是同时评估路径质量、控制成本、稳定性、公平性和部署复杂度。一个方案即使降低了部分流量的时延，如果引入大量路径切换、控制消息或长尾退化，也不能简单判定为更优。The study does not compare mean end-to-end latency alone; it evaluates path quality, control cost, stability, fairness, and deployment complexity together. A scheme that lowers latency for some flows is not automatically better if it introduces many route changes, control messages, or tail-latency regressions.

1.1 主要自变量1.1 Independent variables

  • 拓扑规模、平均度数、链路类型和异构程度；topology size, average degree, link types, and heterogeneity;
  • 采样节点比例、采样周期、事件触发阈值和摘要有效期；sampling-node ratio, sampling period, event thresholds, and summary lifetime;
  • 流量强度、短流比例、突发程度和业务优先级；traffic intensity, short-flow ratio, burstiness, and priority;
  • 链路故障率、降速幅度、恢复时间和观测噪声；link failure rate, slowdown magnitude, recovery time, and observation noise;
  • 路径切换滞回阈值、最短保持时间和批处理窗口。switch hysteresis threshold, minimum hold time, and batching window.

1.2 主要因变量1.2 Dependent variables

  • 平均、P50、P95 与 P99 端到端时延；mean, P50, P95, and P99 end-to-end latency;
  • 吞吐量、丢包率、流完成时间与路径伸长率；throughput, loss rate, flow completion time, and path stretch;
  • 收敛时间、路径切换次数、乱序率和短时环路；convergence time, route changes, reordering rate, and transient loops;
  • 控制消息数量、字节数、CPU 时间、内存和规则更新次数。control messages, bytes, CPU time, memory, and rule updates.

02 · 输入与状态模型02 · Input & state model

统一描述异构链路状态A uniform model of heterogeneous link state

每条链路的观测状态可以表示为带时间戳和置信度的向量，而不是单一权重：Each link's observed state is a timestamped, confidence-tagged vector rather than a single weight:

LinkState = {
  delay_ms,
  loss_rate,
  available_bandwidth_mbps,
  queue_occupancy,
  stability_score,
  observed_at,
  confidence
}

不同数据来源可能具有不同精度。主动探测可以获得端到端时延但会产生额外流量；设备计数器成本低但时间粒度可能有限；仿真器可以提供完整状态，但不等同于真实设备部署。因此，方法中必须记录来源、时间和测量误差。Sources differ in precision. Active probing yields end-to-end delay but adds traffic; device counters are cheap but coarse in time; a simulator provides full state but is not equivalent to a real deployment. The method therefore records source, time, and measurement error.

2.1 状态清洗2.1 State cleaning

候选清洗步骤包括滑动中位数、异常值裁剪、缺失值标记和时间对齐。清洗过程不能只保留“看起来更平滑”的结果，而应比较原始观测与清洗后观测对路径决策的影响。Candidate steps include sliding median, outlier clipping, missing-value flagging, and time alignment. Cleaning must not simply keep whatever "looks smoother"; it compares how raw vs. cleaned observations affect path decisions.

2.2 状态过期2.2 State expiry

任何摘要都必须包含有效期。超过有效期的状态不能继续作为高置信度路径依据；系统应退回更保守的路径或请求重新观测。Every summary carries an expiry. Expired state may not serve as high-confidence path input; the system falls back to a more conservative path or requests re-observation.

03 · CATS 机制03 · The CATS mechanism

分层采样、摘要聚合与变化识别Hierarchical sampling, summarization, change detection

CATS 是 Caelum-Aware Topology Sampling 的名称，不是已标准化协议。机制由三部分组成：CATS stands for Caelum-Aware Topology Sampling; it is not a standardized protocol. It has three parts:

3.1 采样节点选择3.1 Sampling-node selection

候选策略包括按节点度数、介数中心性、流量重要度或分区覆盖率选择采样节点。随机选择作为简单对照，防止复杂策略只是在特定拓扑上过拟合。Candidate strategies select nodes by degree, betweenness, traffic importance, or partition coverage. Random selection is the control, guarding against strategies that merely overfit a topology.

3.2 状态摘要3.2 State summarization

摘要应保留变化方向、幅度、时间戳和置信度。研究将比较固定周期摘要、事件触发摘要以及两者结合的混合方案。Summaries retain change direction, magnitude, timestamp, and confidence. The study compares periodic, event-triggered, and hybrid summarization.

3.3 影响范围估计3.3 Impact-scope estimation

根据故障位置、路径依赖关系和业务优先级确定受影响流集合，避免每次变化都触发全局路径重算。Determines the affected flow set from failure location, path dependencies, and priority — avoiding a global recompute on every change.

3.4 不确定性处理3.4 Handling uncertainty

当观测置信度不足、多个摘要冲突或状态已经过期时，系统优先维持稳定路径，并记录无法做出可靠判断的原因。When confidence is low, summaries conflict, or state has expired, the system favors the stable path and logs why a reliable decision was not possible.

04 · 增量路径计算04 · Incremental path computation

多目标代价与稳定性约束Multi-objective cost with stability constraints

候选路径评分不会固定为某一组未经验证的权重。实验将对不同权重组合进行敏感性分析：Path scoring is not fixed to one unverified weight set; a sensitivity analysis sweeps weight combinations:

path_cost =
    w_delay     * normalized_delay
  + w_loss      * normalized_loss
  + w_congest   * normalized_congestion
  + w_instable  * instability_penalty
  + w_switch    * route_change_penalty

为了避免路由震荡，只有当候选路径改善超过滞回阈值，并且状态置信度、最短保持时间和容量约束均满足时，才允许执行更新。算法还必须检查环路、黑洞和规则安装顺序。To avoid flapping, an update is allowed only when the candidate's improvement exceeds the hysteresis threshold and the confidence, minimum-hold-time, and capacity constraints all hold. The algorithm also checks for loops, black holes, and rule-install ordering.

4.1 触发条件4.1 Trigger conditions

  1. 观测变化超过定义阈值；observed change exceeds a defined threshold;
  2. 变化持续时间超过抖动过滤窗口；the change persists beyond the jitter-filter window;
  3. 受影响路径包含该链路或共享瓶颈；affected paths include the link or a shared bottleneck;
  4. 候选路径改善超过切换成本；the candidate's gain exceeds switching cost;
  5. 更新不会违反容量、环路和策略约束。the update violates no capacity, loop, or policy constraint.
